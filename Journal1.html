Arcade for ArcGIS Online

Arcade is a scripting language for ArcGIS platforms. Common uses of Arcade are for “on-the-fly” calculations, conversions, labeling, and editing pop-ups.

Check out my webmaps below

Example 1

The first tutorial I followed was for converting an attribute field in ArcGIS Online (AGOL). I edited data from a sample data set of global sea temperatures. I converted from Celsius to Fahrenheit. This simple coding proved to be extremely useful for modifying and building data. Frequently, there are struggles of data acquisition in GIS projects, however, Arcade opens up data access. The tutorial also involved changing symbology and adding pop-ups. I appreciated how it’s possible to check your code with the equivalent of a return function with Arcade. The simple coding and “on-the-fly” calculations are a big draw for non-GIS centered people and organizations.

On-the-fly Conversions

Example 2

The second tutorial was for converting integer attribute values to percentages. The data set included total households and households without vehicles. Using Arcade and a simple equation((households without vehicles/total households) * 100), the percentage of households without vehicles was calculated. Then I configured the pop-ups to display the resulting data.

Example 3

The third tutorial was for scaling symbology and adjusting the symbols regardless of map scale. This was applied to hurricane data.

Scaling Symbology

Example 3

The fourth tutorial involved FeatureSets which are Arcade functions that are used in conjunction with feature layers. Therefore, you can do calculations on all the features in a layer. With this example, I calculated the number of trees in each neighborhood, density of trees, and number of specific species of trees. A drawback of FeatureSets is that they are not highly compatible with large datasets and complex expressions because they have to retrieve all the feature data before evaluating. This can impair user experience by running slowly.
